# 设计图：Footprint Map

**职责：** 描述可移植足迹地图的数据来源、通用核心、渲染器、平台适配层与降级输出边界。

## 设计图

```mermaid
flowchart LR
    P[本地 JPEG / HEIC 照片] --> G[足迹生成器]
    M[手动补充的到访点] --> G
    X[后续 GPX / KML] -. v0.2 .-> G

    G --> J[(版本化 GeoJSON)]
    J --> C[通用核心]
    C --> V[数据校验]
    C --> T[时间排序与分段]
    C --> L[直线连接与箭头模型]

    V --> R[互动地图渲染核心]
    T --> R
    L --> R
    A[(Vault 内本地照片)] --> R

    R --> O[Obsidian 适配器]
    R --> H[独立 HTML 查看器]
    R --> F[后续 Markdown 适配器]
    J --> S[静态预览图生成器]

    O --> D[Markdown 内嵌互动地图]
    H --> B[浏览器互动地图]
    F --> W[其他可扩展 Markdown 阅读器]
    S --> I[PNG / JPEG 降级图]
```

## 组件职责

| 组件 | 职责 |
| --- | --- |
| 足迹生成器 | 读取照片拍摄时间与 GPS，合并人工补点，生成稳定 GeoJSON。 |
| 通用核心 | 解析与校验数据，按精确时间排序，生成到访点和相邻连接段。 |
| 互动地图渲染核心 | 渲染底图、照片点位、顺序箭头、弹层、拖动、缩放与视野适配。 |
| Obsidian 适配器 | 识别 `footprint-map` 代码块，解析 Vault 附件路径，处理笔记更新与组件销毁。 |
| 独立 HTML 查看器 | 在普通浏览器中复用同一地图渲染核心。 |
| 静态预览图生成器 | 生成不支持 JavaScript 的 Markdown 解析器可引用的降级图。 |

## 数据边界

- GeoJSON 是点位、时间、照片引用和备注的事实来源。
- Footprint Map 不导入、调用或要求工作区内其他本地项目存在；照片元数据处理由本项目的 TypeScript 模块独立提供。
- 运行时边界为 Obsidian 插件与普通浏览器；不设计 Python/Swift 辅助进程、操作系统后台任务、Apple 原生框架或原生桥接。
- 插件只读取 Vault 内或用户经显式文件导入流程提供的照片，不遍历系统照片库或在 Obsidian 关闭时继续处理。
- EXIF 实现通过项目内的稳定接口隔离第三方解析库，输出只保留标准化后的纬度、经度、拍摄时间与诊断信息。
- 顺序从带时区的精确时间派生，不从“上午/中午/下午”标签推断。
- 连线只表示先后顺序，不请求导航服务，不生成街道路线。
- 平台适配器可解析自身的附件 URL，但不得把平台私有路径写回通用 GeoJSON。

## 故障边界

- 数据文件不存在或整体无效：显示错误和静态降级图。
- 单个点位无效：列出该数据项，继续显示其他有效点位。
- 单张照片不存在或损坏：保留点位与文本，使用照片占位。
- 底图无法访问：保留点线层或进入静态降级，不丢失 Markdown 其他内容。
